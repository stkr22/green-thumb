# AGENTS.md

## Project Structure

Self-hosted plant care tracker (see SPEC.md and docs/, with an overview in README.md):

- `src/greenthumb/` – FastAPI backend: SQLModel (async + aiosqlite), OIDC auth via Zitadel, ntfy notifications
- `alembic/` – database migrations (SQLite; verify with `uv run alembic upgrade head` against a temp SQLite file)
- `tests/` – pytest suite against in-memory SQLite (foreign_keys pragma on)
- `frontend/` – React 19 + TypeScript + Vite + Tailwind 4 + TanStack Query 5 SPA (`cd frontend && npm run build` to type-check and build; `npm run test:e2e` for Playwright e2e in `frontend/e2e/`, which starts its own backend + frontend with an isolated SQLite DB and the dev-login bypass)
- `charts/green-thumb/` – Helm chart (single-instance backend, SQLite on a ReadWriteOnce PVC); published as an OCI artifact and synced by ArgoCD (the ArgoCD Application is owned by the external GitOps repo, not this one)

Project-specific notes:

- Runs on SQLite everywhere. `models/base.py` column types keep PostgreSQL variants (`ARRAY`/`JSONB`) that resolve to `JSON` on SQLite; normalize datetimes with `models.base.ensure_utc` when comparing.
- `db.py` sets WAL + foreign_keys + busy_timeout pragmas per connection. The backend must stay single-instance: the hourly reminder loop runs in-process, so multiple replicas would double-send notifications.
- SQLite cannot `ALTER TABLE ADD CONSTRAINT`: the initial migration inlines the circular `plants.cover_photo_id` FK; `env.py` turns on `render_as_batch` for future autogenerated migrations.
- The ruff `TC` rules are intentionally disabled: FastAPI and SQLModel resolve annotations at runtime.
- Route handlers stay thin; business logic lives in `services/`. Tests authenticate by minting a real session cookie (see `tests/conftest.py`).
- After changing routes/schemas, regenerate the frontend client types: export `openapi.json` (see docs/development.md) and run `npm run generate-types` in `frontend/`.

## Core Commands

Add a dependency to a specific package
```bash
uv add --package inky-image-display pydantic~=2.9.0
```
Apply formatting and Check code quality and Validate types. Always run after the end of a task.
```bash
uv run ruff check --fix . && uv run ruff format . && uv run ty check --fix
```
Install all workspace dependencies
```bash
uv sync --group dev --all-packages
```
Always use uv for any python module/code
```bash
uv run pytest
uv run python3 -c "print(1);"
```
Run the Playwright e2e suite after frontend changes (local only, not in CI). Needs a running API serving the fresh build — see packages/web/README.md.
```bash
cd packages/web && npm run build && npm run test:e2e
```

## Development Guidelines

- **Code Documentation**: Document the *why* a code does what it does not what it does and always check if documentation needs to be updates after making medium to large changes.
- **Enduser and Developer Documentation** is written in Markdown and stored in ./docs folder.
- Validate all external inputs with pydantic or pydantic-settings
- All changes must be tested. If you're not testing your changes, you're not done.
- If **outdated** tests block your implementation suggest changing them.
- Follow existing code style. Check neighboring files for patterns.
- **Clear commit messages**: Explain the *why* a change was done not what the change was, use conventional commits, keep commit message length below 300 characters
